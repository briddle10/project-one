Header (Level 1)

Back to the Top

This block element is the first level of header, which is normally the biggest and most prominent of the text headers.

  • This element has no necessary attributes.

Example: <h1>Lorem ipsum dolor sit amet consectetur adipisicing elit.</h1>

Paragraph

Back to the Top

This is a block element, and the normal element to display text from. It can be stylized just like the six level of headers can.

  • This element has no necessary attributes.

Example: <p>Lorem ipsum dolor sit amet consectetur adipisicing elit.</p>

Unordered List

Back to the Top

This is a block element that creates a list where you can input list items. In this case the bullets are circles and not numbers because it is unordered, indicating the order does not matter.

  • This element has no necessary attributes.

Example: <ul>
List Item
</ul>

Ordered List

Back to the Top

This is a block element that creates a list where you can input list items. In this case the bullets are numbers because it is ordered, indicating the order does matter.

  • This element has no necessary attributes.

Example: <ol>
List Item
</ol>

List Item

Back to the Top

This is an inline element specifically used to display text in an unordered list(ul) or ordered list(ol). Every list item normally has a bullet or number (depending on list type) next to it.

  • This element has no necessary attributes.

Example: <li>Lorem ipsum dolor sit amet consectetur adipisicing elit.</li>
(Only within a list)

Link (a)

Back to the Top

This inline element creates a link to a specific url, id, or other value.

  • Hypertext Reference (href): This attribute specifies the link's destination.

Example: <a href="https://www.google.com/">Click Here For The Googles</a>

Image

Back to the Top

This inline-block element creates an image which is by default a block element with the height and width defaulted to the edges of the image.

  • Source (src): This attribute specifies the path to the image being displayed(locally or with a url) so the web browser can find it.

  • Alternate Text (alt): This attribute is required and is used for a description of the image. This is used for text readers/narrators.

Example: <img src="assets/cat.jpg" alt="A Picture of a Cat">

Hypertext Markup Language

Back to the Top

This element is a tag that tells the web browser that everything it is reading inside of the tag is Hypertext Markup Language.

  • This element has no necessary attributes.

Example: <html>
Content
</html>

Head

Back to the Top

This element is a tag that is necessary to define the page's title, links to style sheets (or other files) and other metadata for the web browser. This section is not displayed on the page. A title is required in the head for HTML files to be readable by the web browser.

  • This element has no necessary attributes.

Example: <head>
<title> The Coolest Web Page Around </title>
</head>

Link (Head Element)

Back to the Top

This element is a tag used to connect between the HTML document and another external file.

  • Relationship (rel): This attribute specifies the relationship between the HTML document and the external file.

  • Hypertext Reference (href): This attribute specifies the link's destination.

Example: <link rel="stylesheet" href="css/main.css">

Title (Head Element)

Back to the Top

This element is a tag that defines a title in the browser tab, provides a title for the page when it is added to "Favorites", and displays a title for the page in search engine results. It is required in the Head of an HTML file.

  • This element has no necessary attributes.

Example: <title>Did We Really Land on the Moon?</title>

Body

Back to the Top

This element is a tag necessary to define what the web browser will display on the page. Ergo everything inside of the tag will be displayed (If coded correctly).

  • This element has no necessary attributes.

Example: <body>
Display Content
</body>

Span

Back to the Top

This element is a tag used to stylize or group inline-elements in an HTML file.

  • This inline element has no necessary attributes.

Example: Lorem ipsum <span id="specialLook">dolor sit amet consectetur</span> adipisicing elit.
(In this example "dolor sit amet consectetur" could be stylized in css by calling the id "specialLook")

Div

Back to the Top

This block element defines a new division or section in an HTML file.

  • This element has no necessary attributes.

Example: <div>
Content
</div>

Header

Back to the Top

This element is a tag used to represent a container at the beginning of the page usually containing header elements an icon or logo and other necessary information or navigation links.

  • This element has no necessary attributes.

Example: <header>
Content
</head>

Nav

Back to the Top

This element is a tag used to define a set of navigation links. Therefore you should have multiple links inside of the nav tag however, they are not required.

  • This element has no necessary attributes.

Example: <nav>
Links
</nav>

Footer

Back to the Top

This element is a tag used to represent a container at the end of the page usually containing copyright, authorship, and contact information or other helpful elements like links to other similar pages for example.

  • This element has no necessary attributes.

Example: <footer>
Content
</footer>

Main

Back to the Top

This element is a tag used to specify the main content of a HTML file. There can only be one main in each HTML file and it is this section that should be different for each page. Ergo outside of the header, footer, or nav tag.

  • This element has no necessary attributes.

Example: <main>
Content
</main>

Section

Back to the Top

This block element is similar to a div tag. It is used to define specific sections of the page.

  • This element has no necessary attributes.

Example: <section>
Content
</section>

Aside

Back to the Top

This element is a tag used to define information (or content) aside from the content that it is placed in. It is often used as a section for supporting information.

  • This element has no necessary attributes.

Example: <aside>
Content
</aside>
